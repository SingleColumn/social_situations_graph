## 1. Interpreted graph design

The graph models two concrete social situations (Situation A: sarcasm; Situation B: genuine encouragement) in which the same words ("Great job") carry different intended meanings depending on observable cues. The design separates:

- **Vocabulary nodes** — abstract, reusable singletons: `Person`, `Tone`, `Expression`, `Context`, `LiteralMeaning`, `IntendedMeaning`.
- **Instance nodes** — per-utterance: `Statement` (one per situation to preserve cardinality rules), `Situation`.
- **Derived signal nodes** — `Signal` nodes that encode the *presence* of a feature (kind + valueId) for pattern matching. These are shared vocabulary-level tokens; situations link to them via `HAS_SIGNAL`.
- **Pattern nodes** — conjunctive rules that `REQUIRES` a set of `Signal` nodes and `PREDICTS` an `IntendedMeaning` with a `probability` property on the relationship.

The key reasoning path is:

```
Situation -[HAS_SIGNAL]-> Signal(s)
Pattern   -[REQUIRES]->   Signal(s)   (all must match)
Pattern   -[PREDICTS {probability}]-> IntendedMeaning
```

When every signal required by a pattern is present in a situation's signal set, the pattern fires and the predicted `IntendedMeaning` (with its probability) is returned.
