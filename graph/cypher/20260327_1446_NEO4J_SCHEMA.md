## 1. Interpreted graph design

The graph models two social situations in which an autistic person must interpret whether a coworker's identical words ("Great job") are sarcastic or genuinely encouraging. The core insight is that **literal meaning is separated from intended meaning**: the same statement can carry different social meanings depending on observable signals (tone, facial expression, context).

**Key structural ideas:**

| Concept | Role |
|---|---|
| `Person` | Speaker in a situation |
| `Statement` | Per-utterance instance node (not unique by text) |
| `LiteralMeaning` | Shared singleton — surface semantics of words |
| `Tone`, `Expression`, `Context` | Observable vocabulary nodes |
| `Signal` | Derived, reusable token encoding *that a feature is present*; bridges vocabulary to pattern matching |
| `Situation` | Concrete scenario instance; links to all observable nodes and its derived signal set |
| `Pattern` | Abstract conjunctive rule: REQUIRES a set of Signals, PREDICTS an IntendedMeaning with a probability |
| `IntendedMeaning` | Social interpretation inferred when a pattern fires |

Pattern matching is conjunctive: a pattern fires for a situation if and only if **every** Signal the pattern REQUIRES is present in the situation's HAS_SIGNAL set. The `probability` property on the PREDICTS relationship encodes the designer-set strength of that interpretation.
